线程安全

首先结构与参数上ConcurrentHashMap与HashMap差别不太大，那么来看看put与get方法，在这两个方法中是如何保证线程安全

数组初始化

关键在于使用sizeCtl变量记录状态，使用CAS方法修改sizeCtl变量，避免多线程重复修改

put方法

在put的时候需要注意，在使用spread方法计算node的hash值时，去掉了符号位，所以后面可以使用node的hash值进行状态的判断，当node的hash >= 0时，使用的是链表存储。

  1. 当table为空的时候进行初始化。
  2. 当索引处为空的时候cas添加。
  3. 当索引处非空的时候，给节点加锁
  4. 添加成功之后调用addCount方法

计数方法

在HashMap中，每次添加之后会++size，但是在多线程的情况下++size这个操作并不安全。于是使用了一个CounterCell[]数组来存储计数。map的实际存储元素数量为baseCount + counterCells中的value和。

  1. 若counterCells非空，调用fullAddCount()修改counterCells
  2. 尝试cas修改baseCount
  3. cas修改失败则调用fullAddCount()修改counterCells
  4. 检查元素总和，是否需要扩容

具体的修改counterCells逻辑在fullAddCount方法中。

扩容机制

在添加元素需要加在链表里或者红黑树中的情况下，则调用addCount方法的时候，最后会检测当前元素数量，与sizeCtl进行比较。前边说了sizeCtl在初始化之后，如果sizeCtl > 0则表示需要扩容的阈值。

总结

在ConcurrentHashMap中，使用了非常精巧的设计，使用非常多的变量来分辨多线程的状态，在大多情况下使用CAS方法，目的就是为了在并发的时候避免使用锁，从而提高并行效率。
