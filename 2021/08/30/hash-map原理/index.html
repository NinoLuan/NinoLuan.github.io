Java HashMap原理学习

简要来说HashMap核心就是用一个数组来存储键值对。

核心逻辑

作为一个容器，hash map的特性就在于

  • 以O(1)的时间复杂度保存key，value的键值对。
  • 以O(1)的时间查询指定key值。在此基础上，就可以实现O(1)时间复杂度的修改删除查询。

hash算法

hash算法，又称为散列算法。定义为：将任意长度的信息转化成一个固定长度的二进制数据。本质上是一个空间映射函数 ，通常来说，我们需要将非常大的输入空间映射到非常小的输出空间（单向，不可逆，不是一一对应）。

常见的实现有MD5，SHA-1，广泛用于加密，签名，验证等。

Java中的hash map也使用了这样的算法，提高查询的效率。对于一个key，首先使用hash算法将key转换为一个hash code值，然后根据hash code值对数组取余就能直接获得数组索引。能够以近乎**O(1)**的时间复杂度从数组中找到指定key的索引。

hash code算法根据不同的对象，可以使用JDK提供的hash code算法，也可以自己重写hash code算法。但是要注意，equals方法与hash code方法有一个逻辑关系。equals方法返回true的时候，hash code的值也应该相等，是equals方法限制了hash code的结果；而hash code相等不代表equals方法要返回true，因为hash code是会冲突的。

扰动函数

在实现了hash code算法之后，hash map也没有直接使用hash code值，而是添加了一层扰动函数。

这层扰动函数的目的是为了让key的hash code能够更加均匀的分布。原因在于，在通过hash code值计算数组索引时，因为数组大小往往很小，取余操作实际上只用到了hash code值的低位信息。添加了扰动函数之后，加入了hash code值的高位信息。（虽然不知道有没有理论证明）经过实际测试，确实大大减小了hash碰撞的概率。

hash冲突

在hash map中，优先使用链表解决hash冲突（在键值对Node结构中保存有next指针），当链表长度超过一定值的时候，将链表优化为红黑树（使用内置的TreeNode结构）。

hash值到数组索引

在确定了hash code的计算之后，使用取模的方法获得数组索引。JDK中设置数组的长度为2的幂次，这样做的话数组索引为0～2 ^ n - 1，数组的最大索引为000011111…，那么对数组长度的取模就刚好转化为了与数组长度2 ^ n - 1的&运算。所以在后面的扩容机制中，数组每次扩容都会变为2倍。

扩容机制

由于使用数组存储键值对，数组的大小是无法动态添加的，所以在存入的对象超过了阈值时会重新创建一个新的数组。数字大小为旧数组的2倍，这是为了后面计算索引、元素迁移更加方便。对于旧的元素会计算索引迁移到新的数组中，链表会以同样的顺序迁移，红黑树会重新检查一次。

线程安全

在多线程的使用环境中，hash map是不安全的。

jdk1.7中，多线程对hash map执行put操作，如果触发resize()，会在链表中发生死循环。并且会出现数据丢失

jdk1.8中，解决了上面这个原因，因为resize()中迁移链表的时候维持了原链表的顺序，不再像1.7中反向了链表。但是多线程下操作hash map依然会有死循环，出现在链表转换为红黑树或者对树进行平衡操作中。多线程中还会出现数据覆盖。
